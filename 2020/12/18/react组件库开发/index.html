react组件库开发

Dec 18 2020

环境配置

  • 安装脚手架工具create-react-app
  • 安装ts
  • 有ts 的加持，代码自动提示补全接口定义的属性，使用时错误属性类型有提示。

    组件测试

    https://jestjs.io/docs/en/getting-started
  • 为什么要组件测试？ 没有组件测试 就要手动创建很多各种不同类型和状态的组件肉眼观察是否符合预期，这样效率低易出错,组件的更改维护困难。测试可以保证代码可以运行并且没有bug.更早的解决bug..重构升级更加容易、可靠。让开发流程更加敏捷。
  • 通用测试框架jest。汲取了之前测试框架的优点，也内置了自己具有的一些特色，同时也是create react app默认的测试框架。

    参看文档介绍一些API的使用

  • react专用测试工具-react-testing-library 用以可以将react 的component（组件）渲染或者说是挂载到测试用例上。react 官方文档首推React Testing Library。作为默认的测试工具。
  • npm run test 运行测试

    什么是Storybook —用于调试预览

    https://storybook.js.org/docs/react/get-started/introduction
    丰富的插件生态系统可帮助您测试，记录和优化故事。您也可以创建一个插件来满足您的工作流程要求
  • create-react-app 入口文件不适合管理组件库
  • 缺少行为追踪和属性调试功能
  • 安装storybook: npx sb init
    运行：yarn storybook

    组件完美开发工具应有的特点

  1. 分开展示各个组件不同属性下的状态
  2. 能追踪组件的行为并且具有属性调试功能
  3. 可以为组件自动生成文档和属性列表

    storybook的一大亮点之：插件系统

  • 插件之文档插件 yarn add @storybook/addon-info -D。两种使用方式：全局使用和stories中使用。

    stoties中使用

    自动生成文档–添加react-docgen-typescript 配合react-docgen-typescript-loader

javascript 模块化发展历史

模块打包—根目录新建tsconfig.build.json (tsconfig配置项：https://www.typescriptlang.org/tsconfig)

npm link 创建软链接，在本地测试组件库.

npm link react-com<组件库名>在需要使用react组件库的项目中创建软链接链接组件库

再创建一个项目 引入组件库 并测试使用

发布包到npm

  1. 将自己编写的包或者命令行程序上传到npm服务器
    npm whoami 检测是否登录，否：npm adduser来注册或者登录账户
    npm config ls 查看登录信息
    如果使用了代理 则需要将代理切换到默认的npm源 npm config set registry https://registry.npmjs.org
  2. 版本号规则 https://semver.org/lang/zh-CN/ https://souche.yuque.com/bggh1p/8961260/ys5fn2
    版本格式：主版本号.次版本号.修订号
  • 主版本号：当你做了不兼容的 API 修改
  • 次版本号：当你做了向下兼容的功能性新增
  • 修订号：当你做了向下兼容的问题修正
  • 先行版本号及版本编译元数据可以加到“主版本号.次版本号.修订号”的后面，作为延伸
